# Prajapati Foundation — Website

Django website for Prajapati Foundation: home, about, education, gallery,
volunteer/join, contact, donation (Razorpay) and an auto-updating news page.

## 1. Setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# now edit .env and fill in real values (see below)

python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Visit `http://127.0.0.1:8000/` and `http://127.0.0.1:8000/admin/`.

## 2. Environment variables (`.env`)

| Variable | Purpose |
|---|---|
| `DJANGO_SECRET_KEY` | Django's cryptographic secret key. Generate one with `python -c "import secrets;print(secrets.token_urlsafe(50))"` |
| `DJANGO_DEBUG` | `True` for local development, `False` in production |
| `DJANGO_ALLOWED_HOSTS` | Comma-separated hostnames, e.g. `example.com,www.example.com` |
| `RAZORPAY_KEY_ID` / `RAZORPAY_KEY_SECRET` | From your [Razorpay dashboard](https://dashboard.razorpay.com/app/keys) |
| `NEWS_AUTO_REFRESH_ENABLED` | `True`/`False` — turns the background news fetch on/off |
| `NEWS_REFRESH_INTERVAL_HOURS` | How stale news must be before an automatic refetch is triggered |

**Never commit your real `.env` file.** It's already in `.gitignore`.

## 3. Donations (Razorpay)

The donation flow is a **secure two-step process**:

1. Browser asks the server to create a Razorpay Order (`/donation/create-order/`)
   — the server decides the amount, the browser cannot fake it.
2. Razorpay Checkout runs using that `order_id`.
3. On success, the browser sends `order_id` + `payment_id` + `signature` back
   to `/donation/save/`, where the server recomputes the HMAC-SHA256
   signature with your secret key and only saves the donation if it matches.

Without `RAZORPAY_KEY_ID`/`RAZORPAY_KEY_SECRET` set, the Donate button is
disabled and a warning banner is shown on the donation page.

## 4. News — auto-fetch

News articles (title, description, image, source link) are pulled from
Google News RSS for Prajapati/Kumhar/pottery-related topics in Hindi.

- **Automatic**: every time someone visits `/news/`, the server checks if
  the newest article is older than `NEWS_REFRESH_INTERVAL_HOURS` (default 6)
  and, if so, fetches new articles in a background thread so the page still
  loads instantly.
- **Manual / cron**: run `python manage.py fetch_news` any time, or put it
  in a cron job / scheduled task for more predictable refresh timing:

  ```
  0 */6 * * * cd /path/to/project && venv/bin/python manage.py fetch_news
  ```

## 5. Media storage — Cloudflare R2 (optional, recommended for production)

By default, uploaded photos (gallery, volunteer applications) are saved to
the local `media/` folder — fine for local development, but **files will be
lost on redeploy** on most hosting platforms (Heroku, Render, Railway, etc.)
since their filesystems are ephemeral.

**Cloudflare R2** fixes this: 10GB storage free, and unlike almost every
other provider, **bandwidth/egress is completely free too** — so serving
photos to visitors never adds to your bill.

### Setup steps

1. Go to [dash.cloudflare.com](https://dash.cloudflare.com) → **R2** → create a bucket
   (e.g. `prajapati-foundation-media`).
2. In the bucket's **Settings → Public Access**, enable the free `r2.dev`
   subdomain (or connect your own custom domain). Copy that URL.
3. Go to **R2 → Manage API Tokens** → create a token with
   **Object Read & Write** permission, scoped to your bucket. Copy the
   Access Key ID, Secret Access Key, and note your Account ID (shown on the
   R2 overview page).
4. In your `.env` file, set:

   ```
   USE_R2_STORAGE=True
   R2_ACCOUNT_ID=<your account id>
   R2_ACCESS_KEY_ID=<from the API token>
   R2_SECRET_ACCESS_KEY=<from the API token>
   R2_BUCKET_NAME=prajapati-foundation-media
   R2_PUBLIC_URL=pub-xxxxxxxxxxxxxxxx.r2.dev
   ```

5. Install the extra dependency (already in `requirements.txt`):

   ```bash
   pip install -r requirements.txt
   ```

6. Restart the server. New uploads (gallery photos, volunteer photos) will
   now go straight to R2 — no model or code changes needed, existing
   `ImageField`s pick up the new storage backend automatically.

If `USE_R2_STORAGE=True` but any `R2_*` variable is missing, Django prints a
warning on startup and uploads will fail until you fix `.env` — it will
never silently fall back and hide the misconfiguration.

**Note:** files already uploaded to local `media/` before switching won't
be automatically moved to R2 — this only affects new uploads. If you need
to migrate existing files, upload them manually to the bucket keeping the
same path structure (`gallery/...`, `volunteers/...`).

## 6. Static files

- `static/` is the **source** — this is what you edit.
- `staticfiles/` is generated by `python manage.py collectstatic` and should
  **never** be hand-edited — it gets overwritten every time you run that
  command. Run it before deploying:

  ```bash
  python manage.py collectstatic
  ```

## 7. Project structure

```
prajapati/            Django project settings, root urls, wsgi/asgi
main/                  App: models, views, forms, admin, news fetcher
main/templates/        base.html + components/ (navbar, footer) + pages/
static/assets/         css/ js/ images/  (source files)
media/                 User-uploaded files (gallery, volunteer photos)
```

## 8. Security notes

- Secrets are never hardcoded — everything sensitive comes from `.env`.
- Razorpay payments are verified server-side (HMAC-SHA256); the browser
  cannot fabricate a "successful" donation.
- File uploads are size- and type-checked (photos: JPG/PNG/WEBP, max 5MB).
- Production (`DJANGO_DEBUG=False`) automatically enables secure cookies,
  HSTS, and SSL redirect.
